Sinds ChatGPT publiekelijk beschikbaar is, experimenteer ik met deze tool als hulpmiddel bij webdevelopment-experimentjes. De afgelopen tijd heb ik gewerkt aan een tooltje voor het bijhouden van mijn persoonlijke aantekeningen. Dat is een webbased tekst-editor geworden, die ik maar 'sNotes' heb genoemd.

Hulp van AI, dankzij hulp van AI

De editor is het resultaat van een reeks verzoeken om code die ik aan ChatGPT heb gesteld. Die heb ik vervolgens zelf met mijn beperkte programmeervaardigheden aan elkaar geknoopt.

Naast het feit dat sNotes is gemaakt met het 'grote taalmodel', is de editor zelf ook voorzien van een vleugje kunstmatige intelligentie. Dankzij een koppeling met de API van OpenAI kan sNotes geselecteerde delen van de tekst corrigeren, verbeteren en vertalen. Dit geeft mij als gebruiker de mogelijkheid om geleidelijk een tekst op te bouwen en per alinea te redigeren.

Prompt engineering

Er kwam wel nog wat prompt engineering aan te pas om ChatGPT te instrueren om niet in volzinnen, maar in JSON-formaat te laten reageren.

"You are an experienced editor. Respond in JSON format. Use the key 'improved_text' for the improved or unchanged text and the key 'openai_comment' for a brief summary of what has changed."

Door deze instructie werd het gedrag vele malen voorspelbaarder en kon ik de verschillende elementen van de reactie in de interface van de tool gebruiken.

Ondanks deze extra instructies gaat het werken met GPT nog steeds gepaard met sporadisch 'onvoorspelbaar gedrag'. Het produceert kromme zinnen of reageert dat gemankeerde tekst toch niet verbeterd hoeft te worden. Veel hangt samen met de versie van het taalmodel waarmee ik nu koppel: 3.5-turbo. Helaas is de wachttijd die versie 4 oplevert niet prettig om mee te redigeren.

Geslaagd experiment

Mijn experiment met sNotes werkt zo goed dat ik het inmiddels privé gebruik bij het maken van verschillende persoonlijke teksten.

Zonder AI had ik dit prototype niet in mijn avonduurtjes kunnen bouwen. Omdat ik weinig programmeer, had ik in de code eindeloos typefouten, misplaatste komma's en de documentatie moeten napluizen. Dat was voor mij altijd de drempel bij complex programmeerwerk - zeker laat in de avond. Dankzij AI had ik de mogelijkheid om met verschillende ideeën te 'spelen' en werd het proces voor mij veel creatiever.

Het experiment heb ik technisch wel simpel gehouden. De functionaliteit in de werkbalk komt uit de open source "Quill" JavaScript-bibliotheek. Dat vond GPT ook wel zo makkelijk. Het opslaan gaat via een paar regels PHP en de koppeling met de OpenAI API is een kwestie van enkele cURL-verzoeken. Dat vond ik dan weer makkelijk, want zo draait het op mijn shared webhosting. Al met al bijft het dus wel een prototype.

Graag zou ik alles werkend publiekelijk delen, maar de API van OpenAI is betaald en de limiet op mijn creditcard is vrij beperkt. Voor wie het interessant vindt staat de code op Github.
